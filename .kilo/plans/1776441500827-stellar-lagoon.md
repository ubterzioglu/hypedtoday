### Phase 3: Prerendering (Static HTML for Crawlers)

**Goal:** Serve static HTML to crawlers at build time so they don't need JS rendering.

**Approach:** Use `vite-plugin-prerender` (uses Puppeteer under the hood) to generate static HTML for key routes at build time.

**Files to modify:**
- `package.json` — Add devDependency `vite-plugin-prerender`
- `vite.config.ts` — Add prerender plugin config

**vite.config.ts addition:**
```ts
import Prerender from 'vite-plugin-prerender';

export default defineConfig({
  plugins: [
    react(),
    Prerender({
      routes: ['/', '/how-it-works', '/contact', '/leaderboard', '/showroom'],
      staticDir: path.join(__dirname, 'dist'),
      renderer: '@prerenderer/renderer-puppeteer',
      postProcess(renderedRoute) {
        // Keep the Vite SPA fallback for client-side routing
        // but serve prerendered HTML for crawlers
      }
    })
  ]
});
```

**Alternative (simpler):** If Puppeteer is too heavy for CI, use `react-snap` or a Vercel Edge Function that injects meta tags per route. Evaluate based on build environment.

> **Note:** Prerendering requires Puppeteer. If the CI/build environment doesn't support it, we'll fall back to a simpler approach: add a Vercel middleware that serves static meta-tag HTML snippets to known bot user-agents.
