# Plan: LinkedIn Admin / LinkStatus Yavaş Yükleme Optimizasyonu

## Sorun

`/adminlink`, `/linkedin` ve `/linkstatus` sayfalarında `linkedin_profiles` tablosundaki veriler çok yavaş yükleniyor.

## Tespit Edilen Performans Sorunları

### 1. LIMITsiz Sorgular (Kritik)
- **`AdminLinkedinProfiles.tsx:47-50`** — Tüm `linkedin_profiles` tablosunu çekiyor, `.limit()` yok.
- **`api.ts:273-277`** (`getLinkedinProfiles()`) — Aynı şekilde LIMIT yok, tüm tablo dönüyor.
- Her iki public sayfa (`/linkedin`, `/linkstatus`) ve admin sayfası (`/adminlink`) bu sorguyu kullanıyor.

### 2. Her Onay/Red Sonrası Tam Refetch (Kritik)
- **`AdminLinkedinProfiles.tsx:79`** — Her approve/reject işleminden sonra `loadProfiles()` ile tüm liste tekrar çekiliyor. 10 kayıt onayla = 10 UPDATE + 10 full-table SELECT = 20 network round-trip.

### 3. Eksik Veritabanı İndexleri
- `linkedin_profiles.linkedin_url` — İndeks yok (dedup ve URL lookup için gerekli)
- `linkedin_profiles.submitted_by` — İndeks yok
- `linkedin_profiles.reviewed_by` — İndeks yok

### 4. RLS Policy Inline Subquery
- `linkedin_profiles` UPDATE policy, `is_admin()` fonksiyonu yerine inline `EXISTS (SELECT 1 FROM profiles WHERE ...)` kullanıyor.

---

## Düzeltme Planı

### Adım 1: DB Migration — Eksik İndexleri Ekle
**Dosya:** `supabase/migrations/20260427_add_linkedin_profiles_indexes.sql`

```sql
CREATE INDEX CONCURRENTLY idx_linkedin_profiles_linkedin_url
  ON linkedin_profiles(linkedin_url);

CREATE INDEX CONCURRENTLY idx_linkedin_profiles_submitted_by
  ON linkedin_profiles(submitted_by) WHERE submitted_by IS NOT NULL;

CREATE INDEX CONCURRENTLY idx_linkedin_profiles_reviewed_by
  ON linkedin_profiles(reviewed_by) WHERE reviewed_by IS NOT NULL;
```

`CONCURRENTLY` ile production'da kilitleme yapılmaz.

### Adım 2: RLS Policy'yi `is_admin()` ile Değiştir
**Dosya:** `supabase/migrations/20260427_optimize_linkedin_profiles_rls.sql`

UPDATE policy'deki inline subquery'yi `public.is_admin(auth.uid())` ile değiştir.

### Adım 3: (İptal) — Sayfalama eklenmeyecek
Kullanıcı tüm verileri görmek istediği için `getLinkedinProfiles()` sorgusuna sayfalama eklenmeyecek. Mevcut yapı korunacak.

### Adım 4: Admin Sayfasında Optimistik Güncelleme
**Dosya:** `src/pages/admin/AdminLinkedinProfiles.tsx`

- `setApprovalStatus` sonrası `loadProfiles()` çağrısını kaldır.
- Yerine local state'te optimistic update yap: ilgili profilin `approval_status`, `reviewed_by`, `reviewed_at` alanlarını anında güncelle.
- Hata durumunda rollback yap.

### Adım 5: (İptal) — Kullanıcı tüm veriyi görmek istiyor
Public sayfalarda ve admin sayfasında filtre/sayfalama eklenmeyecek. Mevcut tüm kayıtlar yüklenecek. Performans iyileştirmesi index + optimistic update + RLS optimizasyonu ile sağlanacak.
