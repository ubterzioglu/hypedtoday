## Faz 2: Sorun Tespiti - Edge Functions Analizi

Asagidaki Edge Functions'lar mevcut ve test edilmeli:

| Edge Function | Method | Auth | Aciklama |
|---|---|---|---|
| `create-post` | POST | requireAuth | Yeni LinkedIn post olustur |
| `claim-task` | POST | requireAuth | Bir gorevi claim et |
| `complete-claim` | POST | requireAuth | Claim'i tamamla |
| `review-claim` | POST | requireAuth | Owner claim'i onayla/reddet |
| `request-limits` | GET | requireAuth | Kullanici limit bilgisi |
| `submit-report` | POST | requireAuth | Rapor gonder |
| `submit-feedback` | POST | Yok (anonim) | Geri bildirim gonder |
| `admin-dashboard` | GET | requireAdmin | Admin panel verileri |
| `admin-actions` | POST | requireAdmin | Admin aksiyonlari |

### Potansiyel Sorun Noktalari (Kod Incelemesi)

1. **CORS origin uyumsuzlugu**: `ALLOWED_ORIGINS` sadece `localhost:5173`, `localhost:3000`, `hyped.today`, `www.hyped.today` iceriyor. Vite dev server `port 8080`'de calisiyor (`vite.config.ts:server.port = 8080`) ama CORS izinlerinde `localhost:8080` YOK. Bu local gelistirmede CORS hatasi verecektir.

2. **Frontend api.ts JSON double-parse**: `apiCall()` fonksiyonu `options.body`'yi once `JSON.parse` ediyor, sonra `supabase.functions.invoke`'a tekrar gonderiyor. `invoke` zaten objeyi JSON.stringify eder - body'yi string olarak gonderiyoruz (`body: JSON.stringify(body)`) ve sonra `maybeJsonBody` ile tekrar parse edip obje olarak veriyoruz. Bu潜在的 olarak sorunlu olabilir.

3. **create-post validation farki**: Frontend `linkedin.com` iceren herhangi bir URL kabul ediyor (`z.string().url().refine(u => u.includes("linkedin.com"))`), ama backend sadece `linkedin.com/posts/` formatini zorunlu kiliyor (`/^https?:\/\/(www\.)?linkedin\.com\/posts\//`). Frontend'de geceni backend reddedebilir.

4. **submit-feedback auth yok**: Bu fonksiyon authentication gerektirmiyor, sadece IP-based rate limiting kullaniyor. Bu kasitli ama test edilmeli.

5. **Validation fonksiyonlari tekrar tanimlanmis**: `supabase/functions/_shared/__tests__/` altindaki test dosyalari validation/response/cors fonksiyonlarini dosya icerisinde tekrar tanimliyor, import etmiyor. Bu test ile uygulama kodunun farklilasmasina neden olabilir.

6. **admin-actions eksik validasyon**: `request_limit_changed` aksiyonunda `payload` validation'u yok - herhangi bir JSON kabul ediyor.

7. **score_events event_type constraint**: DB'de `admin_adjustment_plus/minus` var ama Edge Function `score_adjusted_plus/minus` kullaniyor. Bu fark veritabani constraint violation'a neden olabilir.

## Faz 3: Yeni Testlerin Yazilmasi

### 3.1 Edge Function Unit Testleri (YUKSEK ONCELİK)

Her bir Edge Function icin, Supabase client mock'lanarak unit testler yazilacak:

#### 3.1.1 `create-post` testleri
- Basarili post olusturma (gecerli LinkedIn URL, en az 1 task)
- Gecersiz URL (linkedin.com/posts/ disindaki URL'ler)
- Hicbir task secilmediginde hata
- Rate limit asiminda 429
- Auth olmadan 401
- Suspended kullanici 403
- Request limit asildi (daily/weekly/active/cooldown)

#### 3.1.2 `claim-task` testleri
- Basarili claim olusturma
- Kendi postuna claim yapma (OWN_POST hatasi)
- Aktif postu olmayan kullanicinin claim yapma (GIVE_TO_GET_REQUIRED)
- Duplicate claim (DUPLICATE_CLAIM)
- Task enabled degilse (TASK_NOT_ENABLED)
- Max aktif claim asimi (MAX_CLAIMS)
- Gecersiz UUID post_id

#### 3.1.3 `complete-claim` testleri
- Basarili complete
- Too fast complete (TOO_FAST)
- Comment min length (COMMENT_TOO_SHORT)
- Kendi claim'i degilse 403
- Claim status 'claimed' degilse (INVALID_STATUS)
- Validation hatalari

#### 3.1.4 `review-claim` testleri
- Basarili approve/reject
- Kendi claim'ini review etme (SELF_REVIEW)
- Owner disindakinin review yapma 403
- Pending_review disindaki status ile review (INVALID_STATUS)
- Combo bonus olusturma (like+comment+repost hepsi approved)

#### 3.1.5 `submit-report` testleri
- Basarili rapor gonderme
- Duplicate rapor (DUPLICATE_REPORT)
- Gunluk rapor limiti (10)
- Gecersiz flag_type

#### 3.1.6 `submit-feedback` testleri
- Basarili geri bildirim
- Benzer feedback dedeksiyonu (text_hash)
- Validation hatalari (bos mesaj, max length)

#### 3.1.7 `admin-dashboard` testleri
- Basarili dashboard verisi
- Non-admin 401
- Dashboard istatistikleri dogru hesaplama

#### 3.1.8 `admin-actions` testleri
- Tum action type'lar icin basarili islem
- Gecersiz action_type
- Kritik aksiyonlarda note zorunlulugu (NOTE_REQUIRED)
- user_suspend/unsuspend
- post status degisiklikleri
- claim override
- score adjustment
- flag_reviewed
- global_setting_changed

#### 3.1.9 `request-limits` testleri
- Basarili limit sorgulama
- Auth gereksinimi
- Limit bilgilerinin dogru donmesi

### 3.2 Frontend API Entegrasyon Testleri (ORTA ONCELIK)

#### 3.2.1 `api.ts` testleri (genisletilmis)
- `ensureAuthenticatedSession` - session suresi dolmus ve refresh basarisiz
- `apiCall` - JSON double-parse sorunu testi
- Her bir API metodunun dogru parametrelerle invoke ettigi
- Error response parsing (farkli status kodlari, farkli body formatlari)

#### 3.2.2 `ProjectSubmissionForm` component testleri
- Basarili form gonderimi
- LinkedIn URL validation (frontend vs backend farki)
- Bos form gonderimi
- Network hatasi durumunda toast mesaji

### 3.3 Shared Module Testleri (YUKSEK ONCELIK)

#### 3.3.1 Mevcut testlerdeki sorun duzeltme
- `validation.test.ts`, `response.test.ts`, `cors.test.ts` dosyalarindaki fonksiyonlar import edilmeli, tekrar tanimlanmamali
- `vitest.config.ts` alias'larinin dogru calistigi dogrulanmali

### 3.4 Request Limits Modul Testleri (ORTA ONCELIK)
- `checkRequestLimits` tum limit kontrolleri
- `getUserLimitsSummary` dogru hesaplama
- `logRejection` log yazma
- User overrides ve system limit birlesimi

## Uygulama Sirasi (Strateji: Once Test, Sonra Duzelt)

### Asama 1 - Ortam ve Mevcut Durum Tespiti
1. **Ortam hazirla**: `npm ci`, `.env` kontrol et
2. **Mevcut testleri calistir**: `npm test`, sonuclari kaydet
3. **Lint ve typecheck**: `npm run lint`, TypeScript kontrolleri
4. **Sonuclari raporla**: Hangi testler gecti/hangi testler kaldi

### Asama 2 - Kapsamli Test Yazma
5. **Shared test dosyalarini import ile duzelt**: Tekrar tanimlanan fonksiyonlari import ile degistir
6. **Edge Function unit testlerini yaz**: Her fonksiyon icin mock'lu testler
7. **Frontend API testlerini genislet**: api.ts ve form component testleri
8. **Request limits modul testleri**: checkRequestLimits, getUserLimitsSummary

### Asama 3 - Testleri Calistir ve Hatalari Belirle
9. **Tum yeni testleri calistir**: Hatalari ve bug'lari tespit et
10. **Test sonuclarini ve bulunan bug'lari raporla**

### Asama 4 - Bug Fix'ler
11. **CORS localhost:8080 ekle**: `cors.ts` ve `cors.test.ts`
12. **Validation pattern uyumlastir**: Frontend veya backend LinkedIn URL pattern'ini duzelt
13. **score_events constraint duzelt**: event_type degerlerini uyumlu hale getir
14. **Diger tespit edilen bug'lari duzelt**

### Asama 5 - Final Dogrulama
15. **Tum testleri tekrar calistir**: Hepsinin gectiginden emin ol
16. **Lint + typecheck**: Son kontrol
